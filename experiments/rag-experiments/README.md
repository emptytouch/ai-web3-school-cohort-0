# RAG 实验清单

## 实验 1：协议文档 RAG 问答

**目标**：构建一个最小版本的协议文档问答系统，验证证据链的完整性。

**步骤**：
1. 选择一个官方文档站（如 OpenZeppelin、Uniswap SDK、Ethereum.org 开发者文档）
2. 抓取 10-20 篇页面
3. 按标题结构切 chunk（不按固定字数）
4. 存入向量库（每个 chunk 附带来源 URL、标题路径、更新时间、版本）
5. 用户提问时，返回 answer + sources

**测试用例（三类）**：
- 文档中明确存在的 API 用法 → 期望：准确回答 + 正确 citation
- 文档中不存在的问题 → 期望：拒答 / 说"不确定"
- 旧版本或版本不明确的问题 → 期望：提示需要核对版本

**输出字段**：
```json
{
  "answer": "...",
  "sources": ["url1", "url2"],
  "uncertainties": [...],
  "needs_version_check": true|false
}
```

## 实验 2：RAG 增强交易解释器

**目标**：在已有的"交易风险摘要 Prompt"基础上，加入 RAG 检索层，补充项目背景信息。

**思路**：
- 把合约 ABI、项目文档、历史安全审计报告存入向量库
- 用户输入交易哈希时，先检索相关合约/项目背景
- 把检索结果 + 链上数据一起塞进 Prompt
