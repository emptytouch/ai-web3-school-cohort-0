# Handbook Feedback 流程

> 目录：`handbook-feedback/`
> 文件命名：`YYYY-MM-DD-feedback-[topic].md`

## 反馈类型

在学习 Handbook 过程中，遇到以下情况时创建 feedback：

1. **概念不清楚** — 某个概念解释不够清晰，难以理解
2. **内容过时** — 工具版本、链接、API 已更新
3. **错别字/格式** — 文字错误、排版问题
4. **结构建议** — 章节顺序、内容组织建议
5. **内容补充** — 缺少重要知识点或示例
6. **代码错误** — 示例代码无法运行

## 反馈模板

```markdown
# Feedback: [简短标题]

- **日期**：YYYY-MM-DD
- **Handbook 页面**：[页面 URL]
- **反馈类型**：概念不清楚 / 内容过时 / 错别字 / 结构建议 / 内容补充 / 代码错误
- **问题描述**：（具体描述问题所在）
- **建议改法**：（你认为应该怎么改）
- **来源**：（可选：参考资料、链接）
```

## 反馈流程

1. 学习 Handbook 时发现问题
2. 在 `handbook-feedback/` 目录下创建 feedback 文件
3. 积累到一定数量后，可以提交 PR 到 Handbook 仓库
4. 或在社区中分享讨论
